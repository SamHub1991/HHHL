# HHHL 易用性功能待办

Date: 2026-05-29
Scope: HHHL KMP native client

## 目标

这份文档记录后续能明显提升日常使用体验的功能方向。优先级按用户高频路径、实现收益和对现有架构的贴合度综合排序。

## UI 风格约束

所有涉及界面的功能都应贴合当前产品视觉方向：X-like 信息流、Telegram-like 聊天体验、Apple-like 系统控件和导航质感。

具体要求：
- 时间线、通知、用户帖子、搜索结果继续使用全宽列表行和细分割线，不做独立卡片堆叠。
- 聊天相关功能可以使用气泡、未读角标和轻量会话列表，但聊天样式不要外溢到时间线或设置页。
- 设置、筛选、表单、弹窗、选择器优先使用现有 `HhhlTopBar`、`HhhlActionChip`、`HhhlSegmentedControl`、`HhhlTextInput`、`HhhlStatusRow` 等共享组件。
- 整体视觉应保持精致、现代、克制：信息密度高但不拥挤，状态清晰但不吵，避免大面积装饰性背景和无意义卡片化。
- 新增 UI 要支持主题系统，避免写死颜色；优先使用 `LocalHhhlColors` 中已有语义色。

## 高优先级

### 1. 时间线阅读位置恢复

为每个时间线、频道、通知页记录上次阅读位置和滚动状态。用户返回页面时应能从离开的位置继续阅读。

UI 方向：保持 X-like 信息流结构，不新增重型浮层；可以在顶部或列表内用轻量提示显示“已回到上次位置”。

### 2. 新内容分隔线 / 回到未读位置

在刷新后标记上次已读位置，显示“以下是新内容”或类似分隔线，并提供跳到未读位置的能力。

UI 方向：使用细分隔线加小型标签，视觉重量低于帖子内容；不要用大卡片或弹窗打断阅读。

### 3. 通知筛选

通知页支持按回复、提及、关注、反应、成就、系统消息等类型筛选。信息量大时能快速找到重点。

UI 方向：顶部使用紧凑分段控件或横向 chip，保持 Apple-like 控件质感；通知列表仍保持 X-like 行布局。

### 4. 通知快捷操作

在通知列表中直接回复、加反应、关注回去、标记已读。减少进入详情页的次数。

UI 方向：操作入口应低视觉权重，优先图标按钮或溢出菜单；避免把每条通知做成复杂工具面板。

### 5. 发帖草稿增强

自动保存草稿，并按账号、回复对象、引用对象区分。退出编辑页后返回仍能继续编辑，避免误触丢内容。

UI 方向：Compose 页面保持表单/编辑器风格；草稿恢复提示使用轻量状态行或底部提示，不使用侵入式大弹窗。

### 6. @ 用户、话题、Emoji 自动补全

在发帖输入 `@`、`#`、`:` 时展示候选用户、话题和 Emoji。提升发布速度和准确性。

UI 方向：候选列表应贴近输入框，使用紧凑行项目；头像、用户名、话题热度等信息清晰但不要占用过多高度。

### 7. 图片查看器完善

图片点开后支持缩放、左右滑动、保存、分享、复制链接、查看原图。

UI 方向：使用全屏沉浸式查看器，背景保持深色或主题遮罩；顶部/底部工具栏应克制，操作按钮用图标并支持隐藏。

### 8. 多账号快速切换

支持通过顶部头像、账号菜单或设置入口快速切换账号。不同账号的草稿、阅读位置、通知状态应相互隔离。

UI 方向：账号切换入口应符合 Apple-like shell，列表行展示头像、实例、用户名和当前状态；不要引入独立仪表盘式首页。

## 中优先级

### 9. 统一搜索体验

提供统一搜索入口，覆盖帖子、用户、话题、频道。支持最近搜索和保存搜索。

UI 方向：搜索页应是高密度结果列表；顶部搜索框用共享输入组件，结果用 tabs 或 segmented control 分类。

### 10. 收藏 / 稍后看

提供“稍后看”入口，把想回头看的帖子集中起来。可与实例原生收藏能力并存，也可作为本地增强。

UI 方向：入口应放在帖子操作菜单和个人工具入口中；列表复用 `NoteRow`，避免做成收藏卡片墙。

### 11. 用户快捷操作

在头像或用户名长按菜单中直接关注、静音、拉黑、加入列表、特别关注。

UI 方向：使用现有溢出菜单或底部动作面板；危险操作要有明确颜色和确认，但不要过度弹窗化。

### 12. 时间线密度设置

支持普通、紧凑、超紧凑三档。不同用户对信息流密度偏好差异明显。

UI 方向：设置项使用 Apple-like grouped row；预览区复用真实 `NoteRow` 或轻量示例，不做装饰性大预览卡。

### 13. 关键词过滤和内容偏好

支持静音词、折叠含特定词的帖子、默认隐藏敏感媒体、默认展开或收起 CW。

UI 方向：设置页使用分组列表和紧凑输入；时间线中的被过滤内容以低高度折叠行展示，允许临时展开。

### 14. 离线缓存

断网时仍能查看最近时间线、通知、聊天记录；发帖失败进入重试队列。

UI 方向：离线状态应通过顶部小状态行、列表内状态行或发送队列标记表达；避免全屏错误页覆盖已有缓存内容。

### 15. 分享入口

支持系统分享进 App 发帖；App 内帖子、图片、用户页可分享到系统。

UI 方向：分享动作优先放入标准操作菜单；系统分享进入 Compose 时要保留当前 Compose 视觉结构。

## 小功能但高好感

### 16. 长按菜单

帖子长按支持复制正文、复制链接、分享、翻译、静音线程、举报。图片长按支持保存、复制链接。

UI 方向：使用轻量菜单，不改变列表行常态布局；菜单项图标优先使用现有图标库。

### 17. 手势操作

列表项左滑或右滑支持回复、收藏、加反应等高频动作，并允许在设置中关闭。

UI 方向：手势反馈要克制、明确，避免和系统返回手势冲突；动作颜色使用语义色。

### 18. 加载骨架屏

时间线、通知、用户资料页加载时显示结构化骨架屏，比单纯文字 loading 更稳定。

UI 方向：骨架屏应匹配真实列表行结构和当前主题，不做闪烁过强的动画。

### 19. 失败状态更明确

区分网络失败、登录失效、接口不可用、权限不足，并提供重试、重新登录或设置入口。

UI 方向：优先使用 `HhhlStatusRow`；能局部失败就局部提示，不轻易替换整个页面。

### 20. 应用内更新体验

如果 Android 端已有下载更新逻辑，应完善更新提示、下载进度、安装入口和失败重试。

UI 方向：更新提示符合 Apple-like 系统控件风格，进度清晰但不打断主流程；重要更新可使用对话框，普通更新用设置页或轻量提示。

## 建议首批落地顺序

1. 时间线阅读位置恢复
2. 新内容分隔线 / 回到未读位置
3. 通知筛选
4. 发帖草稿增强
5. 图片查看器完善
6. @ 用户、话题、Emoji 自动补全

这些功能覆盖阅读、通知、发布和媒体四条最高频路径，能最直接提升日常使用体验。

## 2026-05-29 实现记录

本轮已按当前 X-like / Telegram-like / Apple-like 风格约束落地一批高频易用性能力，新增 UI 均保持全宽列表、轻量提示、紧凑控件和主题色语义。

已完成或已有实现支撑：
- 时间线阅读位置恢复：时间线、通知、公告、发现页的高频列表状态在 App 层保留，返回详情/发帖后不重置滚动位置。
- 新内容分隔线 / 回到未读位置：时间线刷新后记录新内容数量和首条未读，列表内显示轻量分隔线，顶部提供“新 N”跳转。
- 通知筛选：通知筛选覆盖全部、提及、互动、回复、引用、成就、特别关心、关注、系统等类型。
- 通知快捷操作：通知行内支持回复、回应、关注回去、查看帖子，并保持低视觉权重 chip 样式。
- 发帖草稿增强：草稿按账号和新帖/回复/引用/频道上下文隔离；恢复后显示轻量提示，用户继续编辑后自动撤掉提示。
- 自动补全：Compose 继续使用已有 @、#、Emoji 候选能力。
- 图片查看器完善：媒体预览支持保存、复制链接、分享、打开原图等操作。
- 多账号快速切换：继续复用已有账号列表和账号级隔离；草稿 key 已纳入账号维度。
- 统一搜索体验：发现页保留帖子、用户、话题、角色、联邦等统一入口；时间线可直接跳到帖子搜索。
- 收藏 / 稍后看：个人入口和列表页文案调整为“稍后看”，继续复用实例收藏能力与 `NoteRow` 信息流列表。
- 时间线密度设置：新增超紧凑密度，和普通/紧凑一起映射到真实 `NoteRow`。
- 关键词过滤和内容偏好：远端静音词接入本地 `NoteRow` 过滤；软静音折叠，硬静音直接隐藏，含引用内容匹配。
- 离线缓存：时间线、通知、聊天继续使用现有缓存；错误状态保留列表内容，不用全屏覆盖。
- 分享入口：Android 支持系统文本分享进 App 并填入发帖页；App 内帖子/媒体已有分享与复制入口。
- 长按/更多菜单：帖子更多菜单已覆盖复制正文、复制链接、分享、静音、举报、删除、加入剪辑等常用动作。
- 用户快捷操作：头像/用户名长按弹出轻量底部动作面板，可直接打开资料、关注、静音、屏蔽、加入列表、切换特别关心；屏蔽保留确认，加入列表复用现有列表选择。
- 手势操作：`NoteRow` 支持右滑回复、左滑稍后看，设置页新增“列表手势”开关；手势反馈使用低视觉权重的语义背景，保持信息流全宽行结构。
- 离线发帖失败重试队列：发帖网络失败会进入失败队列并持久化到草稿存储，Compose 页可重试、载入编辑器或移除，重试成功后自动出队。
- 加载骨架屏：时间线、通知、公告空列表加载时显示贴合真实列表结构的轻量骨架行。
- 失败状态更明确：继续使用 `HhhlStatusRow` 局部错误与重试入口，避免覆盖已有缓存内容。
- 应用内更新体验：Android 更新下载、进度、安装、失败重试逻辑已有增强，并保留设置/系统通知入口。
